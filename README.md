# Рекомендательная система для Яндекс Музыки

## Описание

Прототип персональной рекомендательной системы для музыкального стримингового сервиса. Система использует данные о прослушиваниях ~1.4M пользователей и 1M треков для генерации рекомендаций трёх типов:

- **Топ популярных** — базовый baseline
- **Персональные ALS** — collaborative filtering (implicit)
- **Итоговые** — с ранжирующей моделью CatBoost (3 признака)

Дополнительно рассчитаны похожие треки (i2i) для онлайн-рекомендаций.

## Сервис рекомендаций (Этап 4)

Код сервиса рекомендаций находится в файле `recommendations_service.py`.

### Стратегия смешивания онлайн- и офлайн-рекомендаций

Сервис реализует **гибридную стратегию**, смешивая два типа рекомендаций:

**Офлайн-рекомендации** — предрассчитанные персональные рекомендации, полученные с помощью ALS (collaborative filtering) и переранжированные моделью CatBoost. Загружаются из S3 при старте сервиса. Отражают долгосрочные предпочтения пользователя.

**Онлайн-рекомендации** — формируются в реальном времени на основе последних прослушанных треков (онлайн-история). Для каждого трека из истории подбираются похожие (item-to-item, i2i), рассчитанные через ALS. Отражают текущие интересы пользователя в рамках сессии.

**Алгоритм смешивания (interleaving)**:
1. Если у пользователя **нет персональных офлайн-рекомендаций** — возвращаются топ популярных треков (cold start).
2. Если есть офлайн, но **нет онлайн-истории** — возвращаются только офлайн-рекомендации.
3. Если есть и офлайн, и онлайн — рекомендации **чередуются**: нечётные позиции (1, 3, 5…) заполняются онлайн-рекомендациями, чётные (0, 2, 4…) — офлайн. Дубликаты и уже прослушанные треки исключаются.

Такой подход позволяет:
- Учитывать как долгосрочные, так и краткосрочные предпочтения.
- Быстро реагировать на текущую сессию пользователя.
- Корректно обрабатывать холодный старт (новые пользователи).

### Запуск сервиса

Убедитесь, что переменные окружения S3 установлены (из `.env`):

```bash
export $(grep -E '^(S3_BUCKET_NAME|AWS_ACCESS_KEY_ID|AWS_SECRET_ACCESS_KEY)' .env | xargs)
export S3_ENDPOINT='https://storage.yandexcloud.net'
python3 recommendations_service.py
```

Сервис запустится на `http://localhost:8000`.

### API endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/` | Информация о сервисе (кол-во пользователей, треков, и т.д.) |
| `GET` | `/recommendations/{user_id}?k=10` | Рекомендации для пользователя (смешанные онлайн+офлайн или fallback на топ) |
| `PUT` | `/history/{user_id}?track_id=123` | Добавить трек в онлайн-историю пользователя |
| `GET` | `/history/{user_id}` | Получить онлайн-историю пользователя |

### Примеры запросов

```bash
# Информация о сервисе
curl http://localhost:8000/

# Персональные рекомендации для пользователя 12
curl http://localhost:8000/recommendations/12

# Добавить трек в онлайн-историю
curl -X PUT "http://localhost:8000/history/12?track_id=53404"

# Получить смешанные рекомендации (с учётом онлайн-истории)
curl http://localhost:8000/recommendations/12

# Для неизвестного пользователя (fallback на топ)
curl http://localhost:8000/recommendations/999999999
```

## Тестирование сервиса

Код для тестирования сервиса находится в файле `test_service.py`. Тесты покрывают три сценария:

1. **Пользователь без персональных рекомендаций** — проверяется fallback на топ популярных.
2. **Пользователь с персональными рекомендациями, но без онлайн-истории** — проверяются офлайн-рекомендации.
3. **Пользователь с персональными рекомендациями и онлайн-историей** — проверяется смешивание онлайн и офлайн рекомендаций.

```bash
# Сначала запустите сервис (в отдельном терминале)
python3 recommendations_service.py

# Затем запустите тесты
python3 test_service.py
```

Вывод тестов сохранён в файле `test_service.log`.
